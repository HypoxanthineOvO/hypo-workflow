# 步骤：review_code

## 目标

在测试变绿后审查实现，并评估它与 Prompt 预期产出的匹配程度。

## 指令

1. 检查变更的生产代码，以及有明确理由的测试修改。
2. 检查：
   - 可读性
   - 命名
   - 结构
   - 不必要复杂度
   - 明显 bug
   - 明显安全问题
3. 将实际产出与 Prompt 的 `预期产出` 对比。
4. 如有架构差异，进行总结。

## Diff Score

选择一个评分：

- `1`：完全符合预期产出
- `2`：轻微且可接受的差异
- `3`：明显但合理的差异
- `4`：需要讨论的大差异
- `5`：严重偏离，应停止

## 输出备注

记录到 state 和 log：

- 审查摘要
- 发现的问题
- 架构差异描述
- `diff_score`
