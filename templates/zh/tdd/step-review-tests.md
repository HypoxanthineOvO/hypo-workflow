# 步骤：review_tests

## 目标

在实现开始前，审查当前 Prompt 新增的测试。

## 指令

1. 检查 `write_tests` 中新增的测试。
2. 判断覆盖是否足够：
   - happy path 行为
   - 边界情况
   - 非法输入或失败处理
3. 检查测试命名和结构是否清晰。
4. 如果存在缺口，立即修正测试。
5. 除非测试无法表达，否则不要添加生产实现。

## 审查结论

输出一个结论：

- `通过`
- `需修改`
- `严重问题`

## 输出备注

记录到 state 和 log：

- 审查结论
- 发现的缺口
- 审查期间修改的测试文件
